My extension of the Elemental Naming Scheme

The elemental naming scheme is a cool naming scheme for polytopes. I recognized its potential ever since 2015, as a scheme that treats different symmetry groups as unique mathematical objects with their own families of properties, rather than simply numerically describing the number of sides.

The original elemental naming scheme, as developed by Anton Sherwood and Keiji, only covers convex Wythoffian polytopes. Here, I extend it to cover all nonconvex uniform Wythoffian polytopes, and hopefully eventually all uniform polytopes with reflection across edges.

Note: In accordance with the mainstream opinion in the polytope community, I don't consider snubs to be Wythoffian. I define Wythoffian to be the set of shapes that can be constructed with a generating vertex in a reflecting n-simplex (for rank-n+1 polytopes).

Each shape has a separate name for any Wythoffian symmetry under which it is vertex transitive and has reflection across edges.

The full instructions for naming a shape can be found in this hi.gher.space forum post and a longer spreadsheet of names can be found here.

The executive summary is that a shape is named with a prefix (usually named after some element or material) that indicates its family or the base polytope it is derived from, an infix that indicates the vertex figure type or flag-Wythoffian operation respectively, and a suffix for the dimension.

    Nonconvex

    We use -tomospheno-, instead of -sphenopyri-, for the shapes with trapezoid pyramid vertex figures if they have loop Coxeter-Dynkin diagrams. This is because according to the traversal rules, if the loop is at one end of the diagram, we (almost) always go towards it to get the name.

    Note "repyro", a name for the o3o3o3/2o family which is a retrogradization of the o3o3o3o family. This (and its analogs in other dimensions) is the only non-fissary case I know where two polytopes have the same pattern of ringed nodes in retrograde families that aren't conjugates or quasiconjugates. Thus, we need a new prefix "re-" that only appears in this one situation. Conjugates and quasiconjugates would be handled with words like "quasi-".

    Nonconvex Wythoffians in 4D are the first case that we need new infixes that weren't needed for convex shapes. In particular, -skew- and -cupoli-. We also have some full-symmetrical uses of -podi-. These are for vertex figure types (tuples) that aren't found among convex polychora.
